      Bash Notes (Part 2) - Transforming the input

      Introduction

      In the first part, I introduced the reader to the basic of writing scripts (shebang, comments, parameters, variable and quoting…).

      This second part focuses on processing input parameter or variables of a script: checking parameters, providing default, extracting part via pattern/regexp and calculating.

      The main keywords are: variable expansion, parameter expansion, pattern matching, regular expression and arithmetic evaluation. Searching man bash (Internet…) for this will give wider details and explanations.

      Parameter expansion

      substitution operatormeaning
      ${VAR:-value}Does not set the variable. Return $VAR if set else return value
      ${VAR:=value}Set the variable with value if not set and return $VAR.
      ${VAR:?message}If $VAR is set show its value else display message
      ${VAR:offset:length}If $VAR is set show the $VAR from offset to offset + length

      The last line shows a primitive way of taking a part of the variable without touching it. A typical example of this is taking a part of a date for instance.

      Pattern matching & Regular expressions

      A better way to take some part of a variable (sentence, text, number…) is to use pattern matching.

      Here are the most commonly used parameter expansion using patterns.

      substitution operatormeaning
      ${VAR#pattern}Trim $VAR from the beginning to the first occurence of the pattern (including it) else return the $VAR untouched
      ${VAR##pattern}Trim $VAR from the beginning to the last occurence of the pattern (including it) else return the $VAR untouched
      ${VAR%pattern}Trim $VAR from the end to the first occurence of the pattern (including it) else return the $VAR untouched
      ${VAR%%pattern}Trim $VAR from the end to the last occurence of the pattern (including it) else return the $VAR untouched
      ${#VAR}Variable length

      Patterns are written in a small language allowing to specify sets of strings. It can be used with a lot of bash builtins commands (see: man bash-builtins).

      patternextglobmeaning
      *noMatch zero or more characters
      ?noMatch any single character
      […]noMatch any of the characters in a set
      [:class:]noClass of char (POSIX standard): alnum alpha ascii blank cntrl digit graph lower print punct space upper word xdigit
      ?(patterns)yesMatch zero or one occurrences of the patterns (extglob)
      *(patterns)yesMatch zero or more occurrences of the patterns (extglob)
      +(patterns)yesMatch one or more occurrences of the patterns (extglob)
      @(patterns)yesMatch one occurrence of the patterns (extglob)
      !(patterns)yesMatch anything that doesn’t match one of the patterns (extglob)
      |yeslogical OR between pattern

      (this is inspired by the short but very explainful)

      extglob: Use extending globbing (the pattern language) capabilities. Activation can be checked by shopt extglob. shopt give you the state of your shell options.

      Beside this, some non internal commands are very often in use when writing scripts. And at their heart stands the use of Regular Expressions(RegExp) which is a much more extended language to search for pattern in a text.

      For instance:

      • grep (stands for g/re/p or globally regular expression parsing/printing) is the source to understand your system regular expression standards (thers is many according to the tool. Documentation.
      • sed Stream Editor that make extensive use of RegExp. Documentation.
      • awk Aho - Weinberg - Kernighan a more involved programming language used for parsing. Documentation.

      Although a big subset of this language is common to all of them, there is now many standards of Regular Expression in use depending on the tool/programming language (see: reference on standards). So beware to check on the one in used with your tool.

      Calculating

      These are the practical means to compute from variables:

      • Arithmetic Evaluation (search it in man bash) is internal. Arithmetic expression can be evaluated with:

        $((<arithmetic expression>))

      • bc Basic Calculator is an external command that extend the calculation capabilities (float…). bc can be used for instance:

        echo "<bc programm>" | bc

        or if directly fill a variable with:

        VAR=$(echo "<bc programm>" | bc)

      Exercise

      This script illustrate the use of the knowledge above.

      #!/bin/bash
# This script date a format date X-Y-Z and split it.
# It writes also:
# The day X, The month Y, The year Z

# Usage: ./splitdate 

TODAY=$(date +%d-%m-%y)

DIG=[[:digit:]]
DIG2=$DIG$DIG
DAYMONTH=${TODAY%%-$DIG2}

echo "The day is ${TODAY%%-$DIG2-$DIG2}"
echo "The month is ${DAYMONTH#$DIG2-}"
echo "The year is ${TODAY##$DIG2-$DIG2-}"

# Bonus :-)
# echo "Next year will be $(( ${TODAY##$DIG2-$DIG2-} + 1 ))"
